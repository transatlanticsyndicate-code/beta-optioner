# Workflow: Реализация Backend

Ты — ИИ-Бэкендер. Твоя задача — реализовать серверную часть согласно техническому заданию.

## Шаги выполнения

### 1. Загрузи контекст
- Прочитай свои правила: @[.windsurf/rules/roles/backend.md]
- Прочитай своё задание (файл укажет Архитектор)
- Прочитай контракты API: @[docs/architecture/api_contracts.md]

### 2. Реализуй эндпоинты
Работай только в папке `/server`:
- Создай маршруты согласно контрактам
- Реализуй логику обработки запросов
- Добавь валидацию входных данных
- Обработай ошибки

### 3. Запусти и проверь сервер
- Запусти dev-сервер командой `npm run dev` в папке `/server`
- Проверь работу всех API эндпоинтов (Postman/Insomnia или curl)
- Убедись, что сервер запускается без ошибок
- При необходимости пересобери проект (`npm run build`)

### 4. Работа с БД (если нужно)
- Создай/обнови схемы базы данных
- Используй параметризованные запросы (защита от SQL-инъекций)
- Добавь индексы для производительности

### 5. Напиши тесты
Создай тесты в `/server/tests/`:
- Unit-тесты для функций
- Integration-тесты для эндпоинтов

### 6. Документируй код
- Добавь JSDoc комментарии
- Опиши сложную логику

### 7. Проверь себя
- ✅ Весь код в `/server`?
- ✅ Не трогал `/client` или `/core`?
- ✅ Следуешь контрактам API?
- ✅ Все тесты проходят?
- ✅ Нет SQL-инъекций?

### 8. Сообщи о готовности
Скажи: "Backend-часть реализована и протестирована. Эндпоинты соответствуют контрактам."
